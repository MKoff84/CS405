Overview

This repository contains the major artifacts completed for CS 405: Secure Coding at Southern New Hampshire University. Throughout this course, I developed a comprehensive security policy, implemented secure coding practices, performed vulnerability testing, and created a security presentation demonstrating the application of secure software development principles.

The projects in this portfolio emphasize building secure applications by integrating security throughout the software development lifecycle rather than treating it as a final step. Topics include secure coding standards, risk assessment, DevSecOps automation, encryption, Zero Trust, defense in depth, and vulnerability mitigation.

Learning Reflection

One of the biggest takeaways from this course was learning that security is most effective when it is integrated throughout the software development lifecycle. Rather than treating security as a final checklist, secure coding practices should be incorporated into planning, design, implementation, testing, deployment, and maintenance. Developing the Green Pace Security Policy reinforced the importance of defense in depth, Zero Trust principles, and automated security testing in building resilient software systems.

This course strengthened my ability to identify vulnerabilities, evaluate security risks, implement mitigation strategies, and develop comprehensive security policies that align with industry best practices.
